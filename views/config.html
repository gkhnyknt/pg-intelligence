Configuration Advisor

Live PostgreSQL tuning & performance values (SHOW ALL)

shared_buffers

Primary memory cache for PostgreSQL. Should generally be set to 25% of total system RAM. Too low causes high disk I/O.

effective_cache_size

Helps query planner estimate total available memory (shared_buffers + OS cache). Set to 50%-75% of total RAM for better index usage.

Yükleniyor...

work_mem

Controls memory for ORDER BY, DISTINCT, and Hash Joins. Low values cause disk spills. Recommended: 16MB–128MB per connection.

Yükleniyor...

maintenance_work_mem

Memory used for maintenance tasks (VACUUM, CREATE INDEX, ALTER TABLE). Set it higher than work_mem to speed up these operations.

Yükleniyor...

random_page_cost

Cost of a non-sequentially-fetched disk page. Default is 4.0. If you use fast SSDs/NVMe, set this to 1.1 so planner uses indexes more often.

Yükleniyor...

effective_io_concurrency

Tells PostgreSQL how many concurrent disk I/O operations can be executed. For SSDs, set between 100 to 300 to boost Bitmap Heap Scans.

Yükleniyor...

idle_in_transaction_session_timeout

Kills sessions left 'idle in transaction' after specified time. Prevents bad app code from holding row locks forever and blocking VACUUM.

Yükleniyor...

statement_timeout

Aborts any query that takes longer than the specified milliseconds. Great safety net against runaway queries causing 100% CPU usage.

Yükleniyor...

autovacuum_vacuum_scale_factor

Default 0.2 (20% of rows) is too high for very large tables. Set to 0.01 or 0.05 for tables with millions of rows to clean them more frequently.

Yükleniyor...

checkpoint_completion_target

Controls write spread across checkpoint interval. Default 0.9 is highly recommended to avoid I/O spikes. Pair with higher wal_buffers.

Yükleniyor...

log_min_duration_statement

Logs the duration of any query running longer than this limit (in ms). Essential for observability and capturing slow queries in logs.

Yükleniyor...

max_connections

Don't set this too high (e.g., above 300) as each connection consumes RAM and OS resources. Always use PgBouncer for connection pooling.
